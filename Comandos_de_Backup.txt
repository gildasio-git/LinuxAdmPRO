​Comandos de Backup:



Comando dd​


dd if=<arquivo/dispositivo de origem> of=<arquivo/dispositivo de destino>


        bs= (block size)
        count= (contagem de bs)
        status=progress (mostra o progresso, menos quando usa o count)
        conv=lcase (minúsculo)
        conv=ucase (maiúsculo)
        conv=fsync  (escreve os dados e os metadados antes de liberar o prompt)​



Comando tar:

tar <opções> <origem> <destino>


    c ou --create - criar um novo arquivamento
    v ou --verbose - verbose - mostrar mais informações
    f ou --file - trabalhar com arquivo ao invés de dispositivo de fita
    z ou --gzip - comprimir com gzip
    Z ou --compress - comprimir com o compress
    j ou --bzip2 - comprimir com o bzip2
    J ou --xz - comprimir com o xz
    t ou --list - listar o conteúdo do arquivo.tar
    d ou --diff - Compara entre arquivo.tar e origem
    r ou --append - Adiciona arquivos ao final do arquivo.tar (OBS: não funcioana em compactados)
    x ou --extract - Extrai os arquivos de um arquivo tar
    -C ou --directory - Informa o caminho para o tar comparar ou extrair arquivos
    --skip-old-files - não sobrescreve arquivos
    --keep-newer-files - só sobrescreve os mais antigos que os do arquivo.tar


RSYNC

rsync <opções> <origem> <destino>
        
        -a = -rptlgoD
        -r = recursivo
        -p = preservar permissões
        -l = copiar links simbólicos como links
        -t = preservar a data de modificação do arquivo
        -g = preservar o grupo
        -o = preserver o dono
        -D = preservar arquivos de dispositivos e arquivos especiais
        -v = verbose
        -z = comprimir
        -e = especifica o shell remoto
        --partial = se interrompido continua da parte que parou
        --progress = mostra o progresso
        --delete = apaga no destino se o arquivo não existir na origem
        --ignore-existing = só manda arquivos que não existem no destino, ignora arquivos modificados
        --include = força incluir arquivos ou padrões
         --exclude = exclui arquivos ou padrões



    CENTOS:    
        Pacotes:
                rsync, rsync-daemon
        Para iniciar:
                systemctl start rsyncd
        Para iniciar no boot automaticamente:
                systemctl enable rsyncd
        Para liberar no firewall o acesso ao aplicativo:
                firewall-cmd --permanent --add-service=rsyncd
        Para reler as regras de firewall:
                firewall-cmd --reload
        Para o selinux permitir o acesso do rsync ao chroot:
                setsebool -P rsync_full_access 1
        
        Arquivo de configuração do daemon do rsync:
                /etc/rsyncd.conf
        Exemplo de configuração no arquivo /etc/rsyncd.conf
        
            uid = nobody
            gid = nobody
            use chroot = yes
            max connections = 4
            syslog facility = local5
            pid file = /var/run/rsyncd.pid
            lock file = /var/run/rsync.lock
            log file = /var/log/rsync.log

            [bkp-debian]    
                path = /home/BKP/
                comment = Backup remoto dos servidores
                uid     = vagner
                gid    = vagner
                read only = false
                auth users = bkp01, backup
                secrets file = /etc/senhas-rsync
                hosts allow  = 192.168.1.108
                
        Exemplo de arquivo da opção secrets file:
                usuario:senha
                outrousuario:senha
